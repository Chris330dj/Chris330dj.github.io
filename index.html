写给2022及以后

发表于 2020-11-05

一、写在前面

因为接下来讲的都是我摸索出来的个人经验，不算专业仅供入门。并且注意这只是一种速成学习法，只是用来应付找工作，在此之外还是要多练多学。

且因为个人能力有限，不能囊括所有需要掌握的知识点，建议一边学习一边按个人情况增减。

如果看到这篇的各位有更好的学习方法，欢迎增添修改。

二、推荐书目 & 网站

  • 《JavaScript高级程序设计》 ⭐⭐⭐⭐⭐

    注意买第4版（新编入了一些es6的知识点），这本是入门级的前端书，建议过两遍。

  • 《学习JavaScript数据结构与算法》 ⭐⭐⭐

    讲一些数据结构基础知识，入门够用了但要真研究算法没啥意义（真想看可以买本橙皮的《算法》）。

  • 《你不知道的JavaScript》 ⭐⭐⭐⭐

    进阶书，我还没看完，大概是讲一些JavaScript运行的底层原理。

  • 《图解HTTP》 ⭐⭐⭐

    入门级的HTTP书，（我目前段位）应付面试够用了，其实感觉网络方面的背背面经也行。

  • 《HTML5与CSS3权威指南》 ⭐⭐⭐⭐

    工具书，快速过一遍就行。

  • 掘金 这是一个很优秀的技术社区，没事可以多逛逛。

  • Vue官网

  • React官网

  • 剑指offer - 牛客版

  • 剑指offer - leetcode版

  • git在线练习网站

  • 正则在线练习网站

  • 前端面试题整理 - 前端面试与进阶指南

  • 优秀的github地址 - 前端技能汇总

三、推荐学习流程

3.1 前端三大基础（HTML + CSS + JS）

HTML和CSS部分没有特别说明的，快速过完视频课/书本后多刷刷面经就能应付面试。

我之前收集了一套视频教程，入门级的，按需挑着学学。（B站也有很多学习视频，挑合自己口味的看）

链接：https://pan.baidu.com/s/1g1xOx5mC1zLrEqL0_uH40A 提取码：21im

链接：https://pan.baidu.com/s/1jxpXAahp8ipyALNHcX5SNg 提取码：qt6b

学完这部分可以跟着视频课做做小项目，做一个静态页面出来还是很有成就感的。

然后配合面经查漏补缺，可以看看这个前端面试题整理 - 前端面试与进阶指南，在后续学习中也是这样，虽然技术会了但还是要面向面试学习。

然后是JS，视频课看完一定要配合刷红宝书（JavaScript高级程序设计），推荐是看两遍以上的，第一遍过一下有个概念就行，重要知识点多看，常读常新。

这里就有几个很重要的知识点：DOM，作用域，闭包，原型链，Ajax，跨域，异步请求（Promise/async/await），微任务/宏任务，深拷贝，浅拷贝。 现在不了解没关系，知道这个名词，到时候看书时多留意就行。

JS看完基本概念部分就可以开始同步刷《剑指offer》了，开始的时候每天抽一点时间出来做个一两题，等后续开始笔试了再花时间狂刷。这个时候一般拿到题目后是没有思路的，第一遍可以先想，想不出来看别人怎么做的，多总结不同方法，注意整理。间歇性的也补一下数据结构的知识，栈树链表啥的，可以看《学习JavaScript数据结构与算法》这本书。

JS学完后就可以做一些更有意思的东西，比如一些轮播图啊，菜单栏切换动效啊等等。

3.2 框架学习（Vue / React）

关于框架选择：

现在市场上用的多的，也就Vue和React这两个框架，可以选择其中一个学。我当时选的是Vue框架，原因是官方文档友好，上手也更容易，但后来去工作后又现学了React。

所以我的建议还是学React，这两个框架的一些思想是共通的，学会了React也能快速上手Vue。

学习框架建议是看官方文档，跟着官网教程敲敲代码记记笔记。然后找一个完整的项目跟着做一遍。因为这个项目是你可以写到简历里的项目，建议是不要照抄人家的代码，理解后再自己写。这里我也有几个资料可以借鉴，当然找到自己新的教程更好。

链接：https://pan.baidu.com/s/1-hDbm_Vlh2TloJJ4Kuxehg 提取码：4rw2

（这里分享一个找资源省事省力的办法，去闲鱼搜关键词，例如“前端学习/前端项目”，一般卖的不贵，并且比B站全）

其实还有个框架是Angular，谷歌维护的，国内用的不多。

在这三个框架火起来之前还有个jQuery，现在已经沦落为操作DOM的库了，感兴趣的话看《锋利的jQuery》了解，求职用不到。

关于项目选择：

估计会有人纠结该选什么项目，其实我也提不出太专业的见解，我讲讲我如何选择的吧。

还是以Bilibili为例，搜索前端项目会出现很多不同的项目，如下图。

（我估计挂到github上会看不到图片，好像得翻墙，我再用文字解释一下）

综合排名前列的大概有 谷粒音乐APP，华为荣耀商城，网易考拉官网，品优购电商，阿里百秀，网易严选移动端APP，QQ音乐… 基本就是 音乐APP/电商/后台管理系统。

  1. 根据自己的技术栈选择，或者你想学个大概逻辑框架，再用自己的技术栈实现也行。（比如我当时学的Vue，就找了个Vue开发的音乐APP项目）

  2. 注意时长，只有十几集每集十几分钟的那种不要学，基本就是做了个皮毛，面试的时候没法让面试官从中找亮点。

  3. 看最终呈现结果，先看看最后的效果你喜不喜欢，要是你觉得，哇哦这很酷，那就做这个项目吧。

  4. 是否提供项目源码，有时候仅凭文字和视频可能会出现一些奇奇怪怪的问题，这时候对照源码就会轻松不少（但不是ctrl+c/v🙅‍♀️）。还有个理由是你可以偷代码里的一些css参数（比如组件宽高多少px，选什么颜色等等，我觉得这些无足轻重）

    我选择音乐项目的一个重要原因是网易云音乐 API，这个项目相当于一个本地后台，接口文档什么的都很全，我要做的只是本地启起来后向后台发送请求。

  5. 扫一遍项目知识点，简单来说就是我能从这个项目教程里偷到哪些可以应对面试官“你在项目中遇到了哪些难点，以及怎么解决”这个问题的知识点。我认为一个前端项目不可或缺的知识点有，利用ajax/axios进行前后端通信，做一些简单的组件封装（轮播图，瀑布流，滚动列表等等），鼠标键盘事件（web端）或者手势事件（移动端），对数据格式的加工处理等等。

    比如一些仿商城/官网的项目，很多都只是简单做一个静态项目，这种项目写在简历上简要概括都写不了几句话。电商后台的话前几年好像就烂大街了，前端后端都在做，建议做一些新鲜的东西。

    我同学做了一个仿某旅游APP的项目，里面会学到一些调地图api接口的工作，可供参考。

3.4 框架精进

框架不但要会用，一些原理也得要会，需要看源码。一般面试问的也不深，翻来覆去就那几个问题，建议对着面经学。

3.5 附加

以上学完可以再学点别的，如

git使用，微信小程序，Node.js，TypeScript，React Native，Webpack，RESTFul Api，数据库… 啥的。

也可以对着目标公司的JD学，比如我随便找个一家公司前端开发的JD，

  1. 熟练掌握各种前端技术，包括HTML/CSS/JavaScriptde等；

  2. 熟悉W3C、ECMAScript、ES6等相关技术标准

  3. 至少使用或者深入了解vue、react等主流开发框架

  4. 了解nodejs，了解 Grunt、Gulp、Webpack其中任意一项构建工具的使用和配置

  5. 熟悉linux基本命令

  6. 具有高度抽象能力、思路清晰、善于思考，能独立分析和解决问题；责任心强、耐心，具有良好的团队合作精神和承受压力的能力

以及关注几个分享前端知识的公众号，逛逛社区，逛逛牛客讨论区。

学到这个阶段自然会有一些自己对这个行业的了解，这里就自己挖掘自己选择吧。

以我的水平最多也就教这么多了，愿各位有更好的出路，愿咱们师门的就业状况从下届开始好转💪。

四、重要时间点

实习：一般在3-5月份，就是过完年就得开始准备了，并且因为实习也有提前批的概念，早点准备也没差的😂。找实习尽量是找能转正的，认真干转正了秋招也不会太慌。如果没有的话，干个2-3月就溜，实习的时候多摸鱼学知识，不要耽误秋招。要是还不行的话，找个日常实习（说要干6个月以上的那种），到时候实习一段时间就溜，混个实习证明。

秋招： 关注各大招聘公众号，我们学校办的”柚子求职帮“就很不错，大概从8月份开始就陆陆续续会有一些招聘信息，有些公司提前批也许更早。这里注意，有的公司说提前批失败不影响正式批投递，实际上会影响的。但提前批可以免笔试，还能提前收到offer早点结束秋招，所以如果准备好就早点冲吧！
